# auto-punch

Standalone CLI for scheduled Apollo HR clock-in / clock-out on weekdays.

Replaces the apollo-skill-dependent `auto-punch` directory with a self-contained
Python package — no external script paths to wire, just `pipx install`.

## Install

```bash
pipx install ~/work/auto-punch
```

## First-time setup

```bash
auto-punch login
# Prompts for company code / username / password.
# Runs OAuth + BPM SSO handshake, writes ~/.config/auto-punch/.env.

auto-punch run --type in --dry-run    # verify decision logic
auto-punch run --type out --dry-run
auto-punch status                      # today + 5 upcoming weekdays
```

## Scheduling via launchd

```bash
auto-punch install-launchd     # writes morning + evening plists, runs smoke test
auto-punch uninstall-launchd   # reverses the above
```

The plists fire at 9:00 / 18:00 each day. Each invocation:
- Skips weekends, TW public holidays, days marked as leave, and slots already
  punched (logged, no notification).
- Computes a deterministic per-day offset in `[-10, +20]` minutes so punch
  times look natural.
- Sleeps to the target time, then calls Apollo's punch API.
- Notifies via macOS notification on success or failure.

## Config

`~/.config/auto-punch/.env`:

| Key | Set by |
|---|---|
| `APOLLO_COMPANY_CODE` / `APOLLO_USERNAME` / `APOLLO_PASSWORD` | `auto-punch login` |
| `APOLLO_COOKIES` / `APOLLO_COOKIES_UPDATED_AT` | login + auto-refresh on auth error |
| `AUTO_PUNCH_SECRET` | random hex, generated by login |
| `AUTO_PUNCH_LOG` | path to JSONL event log |

Override config path with `AUTO_PUNCH_CONFIG=...` env var.
